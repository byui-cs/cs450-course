Neural Network Part 2

Objective

Understand Neural Networks, in particular, back-propagation for multi-layer perceptron networks.

Please note that you have 3 weeks to implement and experiment with this algorithm. This is because it can be a challenge! This is likely the most challenging individual programming assignment we will do this semester. Do not put this off!

The purpose of this assignment is to make sure you are on track to be successful with this assignment.

Instructions

As stated in last week's assignment, you can choose between implementing a neural network or doing in-depth experimenting with a library implementation. Depending on your choice, please complete the following steps in one category or the other. (If you previously started down one path, but have decided to switch to the other option, you should complete last week's requirements for that option, and then complete this week's.)

Implementation Option

If you have chosen to implement a neural network yourself, by the end of this week your goal is to be able to construct a multi-layer neural network of arbitrary configuration (i.e., number of layers, and number of nodes per layer) and use that network to make classifications on a dataset. At this point, it will be at best random guessing, because we won't have the backpropagation/learning piece finished yet. But you should be able to run it and get results.

At a more specific level, you need to have at least the following completed this week:

  1. Build a classifier with an arbitrary number of layers, and an arbitrary number of nodes in each layer.

    • For example: 2 Layers (1 hidden, 1 output), with 4 nodes in the hidden layer, and 3 output nodes.

    • Or 3 Layers (2 hidden, 1 output), with 2 nodes in the first, 3 in the second, 1 in the third.

    • Or 2 Layers (1 hidden, 1 output) with 8 nodes in the hidden layer, and 2 in the output.

    • Etc.

  2. The number of weights for the first layer should be determined by the number of input attributes.

  3. Biases are present at every layer.

  4. You should be able to complete the feed-forward portion of the algorithm, looping through each node of each layer to produce values at the output layer.

  5. For your activation function, use the sigmoid function: f(x) = 1 / (1 + e^-x)

  6. You should be able to classify an instance using your network. Please note that at this point, you will not have implemented any weight updates, so your network will essentially be randomly computing an answer, but it should be able to classify a given instance.

  7. Using the classification described, you should be able to classify a complete dataset and calculate the accuracy. (E.g., try it on the iris dataset)

Please note that this week, we are still not implementing the error calculation and back-propagation. We will finish that up next week and then have some time to experiment with different datasets and network configurations. If you can get that part done this week as well, you will be better off, but the minimum standard for this week is to complete the feed forward network as described above.

Experimentation Option

If you have selected the experimentation option, this week your task is to use a library implementation to make predictions on several different datasets of different characteristics. Then, next week, you will pick one or two of these datasets and experiment with various hyper-parameter combinations.

In particular, you should use and make predictions on at least four different datasets. Then, among your datasets you should make sure that you have worked with at least the following characteristics in some form:

  1. Text values

  2. A significant amount of columns

  3. A significant amount of rows

  4. Binary classification (only two possible values)

  5. Classification with more than two target values

  6. Regression (numeric target values)

  7. Look at different measures of success for your datasets (more than just accuracy)

This does not mean that you must use a different dataset for each of the above characteristics, nor does it mean that one dataset needs to cover all of them. You may have one dataset that covers a few, and other that covers others.

In the end, your goal is to have prepared and made predictions on a number of different datasets so that you can choose a "hard" one or two from among them that you will focus on next week.

It is your task to find these datasets. One potential resource to consider is the UCI Machine Learning Repository from which many of our datasets this semester have come.
